Действуй как опытный backend-разработчик на Node.js. Твоя задача — создать серверное приложение (REST API) для управления списком пользователей с подключением документоориентированной базы данных MongoDB[cite: 744, 745]. Используй фреймворк Express и библиотеку Mongoose[cite: 643, 647].

Шаг 1: Инициализация и настройка
Установи необходимые зависимости (express, mongoose)[cite: 647]. Настрой подключение к локальной базе данных MongoDB[cite: 649, 650]. 

Шаг 2: Создание модели (Схемы БД)
Сущность Пользователь должна включать следующий строго определенный набор полей[cite: 746]:
- ID: Integer (Уникальный идентификатор пользователя)[cite: 747]. Примечание: так как MongoDB по умолчанию использует строковый ObjectId, реализуй логику автоинкремента или кастомного присвоения числового ID при создании.
- first_name: String (Имя пользователя)[cite: 747].
- last_name: String (Фамилия пользователя)[cite: 747].
- age: Integer (Возраст пользователя)[cite: 747].
- created_at: Number (Время создания пользователя, должно сохраняться в формате unix timestamp)[cite: 747].
- updated_at: Number (Время обновления пользователя, должно сохраняться в формате unix timestamp)[cite: 747].

Шаг 3: Маршруты и контроллеры (CRUD)
Обязательно свяжи логику каждого маршрута напрямую с базой данных MongoDB (через методы Mongoose, такие как .find, .save, .findByIdAndUpdate и т.д.)[cite: 751]. Реализуй следующий перечень энд-поинтов:
- POST /api/users : Создание нового пользователя[cite: 749].
- GET /api/users : Получение списка пользователей[cite: 749].
- GET /api/users/:id : Получение конкретного пользователя по его числовому ID[cite: 749].
- PATCH /api/users/:id : Обновление информации пользователя (возвращает обновленную запись)[cite: 749].
- DELETE /api/users/:id : Удаление пользователя[cite: 749].

Шаг 4: Итоговый результат
1. Сгенерируй весь необходимый код приложения (точку входа, подключение к БД, определение Mongoose-схемы/модели и роуты).
2. Реализуй автоматическое обновление поля updated_at при PATCH-запросах.
3. Предоставь краткую инструкцию по запуску проекта и проверке работоспособности базы данных.
