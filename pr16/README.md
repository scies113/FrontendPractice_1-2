# Практическая работа №16: WebSocket + Push

## Описание
Реализация real-time коммуникации через **WebSocket** (Socket.io) и отправка **Push-уведомлений** (Web Push API). Сервер на Node.js + Express принимает подключения клиентов, рассылает события о новых задачах и отправляет push-уведомления даже при закрытой вкладке.

**WebSocket** — протокол постоянного двустороннего соединения. Сервер может отправлять данные клиенту в любой момент без запроса. Идеален для чатов, лент активности, онлайн-игр.

**Push-уведомления** — позволяют отправлять сообщения даже при закрытом приложении через Web Push API и VAPID-ключи.


## Что реализовано
- **Сервер (Node.js + Express + Socket.io + web-push)** — порт 3001, обработка подключений, рассылка событий, отправка push-уведомлений
- **WebSocket (Socket.io)** — двустороннее соединение, событие `newTask` → `taskAdded`, всплывающие уведомления между вкладками в реальном времени
- **Push-уведомления (Web Push API)** — VAPID-ключи, подписка/отписка через эндпоинты `/subscribe` и `/unsubscribe`, обработка push в Service Worker
- **Индикатор сети (Онлайн/Офлайн)** — зелёная/красная точка в шапке, реагирует на события `navigator.onLine`, `online`, `offline`
- **Service Worker** — кэширование статики (Cache First), кэширование динамики (Network First), обработка push-событий, показ системных уведомлений, клик по уведомлению открывает приложение
- **App Shell** — каркас приложения с вкладками «Главная» и «О приложении»
- **LocalStorage** — хранение заметок между сессиями
- **SCSS** — переменные, миксины кнопок, анимации (slideIn, slideInRight), адаптив для мобильных

## Технологии
- HTML5 + CSS3 (SCSS/SASS)
- Vanilla JavaScript (ES6+)
- Socket.io 4.x (WebSocket)
- Web Push API (VAPID)
- Service Worker API (Cache API, Push API, Notifications API)
- Node.js + Express (бэкенд)
- LocalStorage API
- npm (скрипты сборки и запуска)



## Запуск

### 1. Установка зависимостей
```bash
cd pr16
npm install
```

Установятся: `express`, `socket.io`, `web-push`, `cors`, `body-parser`

### 2. Запуск сервера
```bash
npm run dev
```

Сервер запустится на `http://localhost:3001`. В терминале появится:
```
Сервер запущен на http://localhost:3001
```

### 3. Открыть приложение
Откройте браузер по адресу **`http://localhost:3001/`**

### 4. Тестирование WebSocket
1. Откройте **две вкладки** с `http://localhost:3001/`
2. В первой вкладке добавьте заметку
3. Во второй вкладке появится **всплывающее уведомление** справа вверху: `📨 Новая задача: ...`
4. Уведомление исчезнет через 3 секунды

### 5. Тестирование Push-уведомлений
1. Прокрутите страницу вниз до футера
2. Нажмите **«🔔 Включить уведомления»**
3. Браузер запросит разрешение — нажмите **«Разрешить»**
4. Кнопка сменится на **«🔕 Отключить уведомления»**
5. Добавьте заметку — придёт **системное push-уведомление** (как обычное уведомление ОС)
