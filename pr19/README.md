# Практическая работа №19: User Management API

Это серверное приложение на Node.js с использованием Express и Sequelize для управления пользователями в базе данных PostgreSQL.

## Инструкция по запуску

1. **Предварительные требования**:
   - Установленный Node.js.
   - Установленный PostgreSQL.
   - Созданная база данных (по умолчанию `postgres`).

2. **Настройка переменных окружения**:
   - Откройте файл `.env` в корне папки `pr19`.
   - Укажите ваши данные для подключения к PostgreSQL (`DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`).

3. **Установка зависимостей**:
   ```bash
   cd pr19
   npm install
   ```

4. **Запуск приложения**:
   ```bash
   npm start
   ```
   Сервер запустится на порту 3000 (или указанном в `.env`). База данных синхронизируется автоматически.

## API Эндпоинты

- `POST /api/users` — Создать пользователя (body: `{ "first_name": "Ivan", "last_name": "Ivanov", "age": 25 }`).
- `GET /api/users` — Получить список всех пользователей.
- `GET /api/users/:id` — Получить пользователя по ID.
- `PATCH /api/users/:id` — Обновить данные пользователя.
- `DELETE /api/users/:id` — Удалить пользователя.

## Подсказки для защиты (FAQ)

1.  **Что такое Sequelize?**
    - Это ORM (Object-Relational Mapping) для Node.js. Она позволяет работать с базой данных через объекты JavaScript, не написав ни одной строки на SQL.

2.  **Зачем нужен файл `.env`?**
    - Для хранения конфигурационных данных и секретов (пароли, ключи API, порты) отдельно от кода. Это повышает безопасность и гибкость настройки приложения.

3.  **Как реализована синхронизация с БД?**
    - Используется метод `sequelize.sync({ alter: true })`. Он проверяет текущее состояние таблиц в БД и приводит их в соответствие с моделями, созданными в коде.

4.  **Какая разница между PUT и PATCH?**
    - В данной работе используется `PATCH`, так как он предназначен для частичного обновления ресурса. `PUT` обычно предполагает полную замену объекта.

5.  **Как обрабатываются ошибки?**
    - В контроллерах используется блок `try...catch`. Если база данных возвращает ошибку (например, невалидные данные), сервер отправляет статус 400 или 500 с описанием ошибки в формате JSON.

6.  **Что такое Middleware `express.json()`?**
    - Это встроенный посредник в Express, который парсит входящие запросы с JSON-телом и делает эти данные доступными в `req.body`.
