# Практическая работа №15: App Shell

## Описание
Внедрение архитектуры **App Shell** (каркас приложения). App Shell — минимальный набор HTML/CSS/JS, который кэшируется при первом посещении и загружается мгновенно. Динамический контент (страницы «Главная» и «О приложении») подгружается внутрь контейнера.

## Что реализовано
- **App Shell архитектура** — статический каркас (`index.html`) + динамический контент (`/content/*.html`)
- **Две страницы**: «Главная» (заметки) и «О приложении» (информация о технологиях)
- **Два кэша в Service Worker**:
  - `notes-cache-v3` — статические ресурсы (Cache First)
  - `dynamic-content-v1` — динамические страницы (Network First)
- **LocalStorage** — хранение заметок между сессиями
- **SCSS** — переменные, миксины, вложенность, анимации, адаптив

## Технологии
- HTML5 + CSS3 (SCSS/SASS)
- Vanilla JavaScript (ES6+)
- App Shell Architecture
- Service Worker API + Cache API
- LocalStorage API
- PWA (Progressive Web App)
- npm (скрипты сборки)

## Запуск

### Обычный режим (HTTP, для разработки)
1. Переходим в папку проекта:
   ```
   cd pr15
   ```
2. Устанавливаем зависимости:
   ```
   npm install
   ```
3. Запускаем сервер:
   ```
   npm run dev
   ```
4. Открываем `http://127.0.0.1:8080/`

### Режим HTTPS (требуется по заданию)

#### Шаг 1: Установка mkcert

**Windows:**
```
choco install mkcert(либо просто скачиваем .exe-файл с гитхаба и закидываем в system32)
```

#### Шаг 2: Генерация сертификатов

В папке(pr15) выполните:
```
mkcert -install          # один раз — доверять локальным сертификатам
mkcert localhost 127.0.0.1 ::1   # создаст localhost.pem и localhost-key.pem
```

> ⚠️ Файлы сертификатов (`*.pem`) добавлены в `.gitignore` — **не коммитьте их!**

#### Шаг 3: Запуск HTTPS-сервера

```
npm run dev:https
```

Откройте `https://localhost:3000/`
