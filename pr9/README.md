# Практическая работа №9: Refresh Tokens

## Описание
реализация механизма обновления токенов доступа без повторного ввода пароля.
используется ротация refresh-токенов для повышения безопасности.

## Особенности
- Access токен живет 1 минуту (для демонстрации).
- Refresh токен живет 2.5 минуты. (также для демонстарции)
- при использовании refresh-токена старый уничтожается, выдается новый.

## Запуск
1. переходим в папку back(cd back)
2. npm install(зависимости)
3. npm run dev(запуск)
4. открыть http://localhost:3000/api-docs

## Сценарий теста
1. POST /api/auth/register - регистрируем пользователя
2. POST /api/auth/login - входим в систему(логинимся)
3. после входа в систему, нам дает 2 токена (Access и Refresh)
4. GET /api/auth/me - проверяем жизнь токена(Access)
5. Подождать истечения Access .
6. вызываем POST /api/auth/refresh с полученным ранее Refresh токеном.
4. получаем новую пару токенов.
5. пытаемся использовать старый Refresh токен приводит к ошибке 401.
